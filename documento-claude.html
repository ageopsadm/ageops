Contexto

1. Quem somos e de onde surgiu o AGE OPS

A AGE é uma produtora audiovisual brasileira especializada em vídeos, videoclipes, eventos ao vivo, content days, publicidade e podcasts. Fundada em 2021, opera desde então com uma equipe enxuta de criadores e freelancers, atendendo clientes que vão de artistas da música (Universal Music, MC Bill, Jay P) a empresas e marcas (Cobertec, Escola de Planejados, Casa 46).

Durante 4 anos de operação, o controle de negócios era feito por planilhas Excel dispersas: uma por ano, sem padronização, sem integração, sem visibilidade em tempo real. Informações como margem de lucro, custo por projeto, NPS de clientes e produtividade da equipe existiam "espalhadas" e dependiam de consolidação manual.

O AGE OPS nasceu da necessidade real do Gustavo (fundador) de operar sua empresa de qualquer lugar, com visibilidade total, e delegar com confiança para a equipe. A ideia central: se o sistema sabe de tudo, qualquer colaborador pode tomar decisões corretas sem depender do fundador presente.

"O diferencial não é a ferramenta — é a adaptação personalizada para as necessidades reais do negócio de cada pessoa."

O projeto foi desenvolvido inteiramente dentro da plataforma Genspark, usando HTML, CSS e JavaScript puro com uma API RESTful de banco de dados. Sem backend proprietário, sem servidor dedicado — 100% operacional via browser.

Descrição do Produto

2. O que é o AGE OPS

O AGE OPS é um sistema operacional web completo para gestão de produtoras e agências criativas. Ele centraliza em uma única aplicação tudo que uma empresa de serviços criativos precisa para operar: projetos, financeiro, equipe, clientes, NPS, orçamentos e estratégia.

O sistema opera com uma única URL — sem instalação, sem servidor, sem app. Qualquer membro da equipe acessa pelo browser com login e senha, com permissões por papel (Admin vs Equipe). O fundador pode estar em qualquer cidade e ter visão completa em tempo real.

Filosofia de Design

  • Fonte única de verdade: todos os módulos leem dos mesmos dados, evitando divergências entre "a planilha do financeiro" e "a planilha do projeto".
  • Integração bidirecional: alterar o custo no Fechamento atualiza automaticamente o Projeto, que atualiza o Dashboard, que atualiza o Histórico.
  • Operação offline-first: dados críticos (planilha 2026) ficam em memória, garantindo performance mesmo com latência de rede.
  • Delegação real: cada membro da equipe enxerga exatamente o que precisa para tomar decisões sem escalar para o fundador.

Fluxo de integração entre módulos

Um dos aspectos mais sofisticados do sistema é que todos os módulos se comunicam automaticamente via um barramento de eventos interno (os-sync.js):

/* Exemplo: salvar um gasto no Fechamento dispara uma cadeia */

FECHAMENTO salva gasto
  → age_projects.cost atualizado (custo real)
  → age_projects.profit_pct recalculado
  → Dashboard recarrega KPIs com margem real
  → age_hist_projects sincronizado (se projeto concluído)
  → age_clients_db recalcula métricas do cliente

KANBAN move card para "No Ar"
  → age_projects.status = "concluido"
  → Dashboard atualiza contagem de projetos ativos
  → Histórico inclui projeto nas métricas anuais

Decisões de arquitetura relevantes

  • Dados críticos em memória (JS const): a planilha 2026 (PROJ_2026_SHEET) fica como constante JavaScript. Isso garante performance e resiliência a falhas de rede, mas exige sincronização manual com o banco.
  • Edição inline com lock de concorrência: cada célula da tabela de projetos tem um sistema de "lock" (_ieSaving Set) que evita chamadas duplicadas ao banco durante digitação rápida.
  • Deduplicação automática: o sistema detecta e remove registros duplicados no histórico que podem surgir de re-importações de planilhas.
  • ID preservation no Sync: o botão Sync nunca deleta e recria registros (o que quebraria vínculos com fechamentos). Em vez disso, faz UPDATE quando o registro existe, preservando o UUID original.
  • NPS completamente anônimo: o sistema técnico garante que nem mesmo o banco sabe qual cliente respondeu o NPS — o token de acesso não carrega informações pessoais.
Diferenciais

6. Diferenciais do Projeto

🎯 Personalização Real, Não Genérica

Enquanto ferramentas como Asana, Monday ou Notion são horizontais (servem para qualquer empresa), o AGE OPS foi construído especificamente para o fluxo de uma produtora audiovisual: catálogo de pacotes com equipe e custo base, etapas do Kanban refletindo o fluxo real de produção, métricas de NPS por etapa (não só ao final), histórico de artistas vs. marcas como segmentos distintos.

🔗 Integração Total Entre Módulos

Não é uma coleção de ferramentas — é um sistema. Alterar o custo num fechamento reflete imediatamente no dashboard, no histórico e no cliente. Isso elimina o problema clássico de "minha planilha de projetos diz uma coisa, meu financeiro diz outra".

📍 Operação Remota Real

O fundador pode estar em qualquer lugar e a equipe opera de forma autônoma. Cada colaborador entra com seu login e vê exatamente o que precisa. Sem dependência de estar no escritório ou de "perguntar para o Gustavo".

📊 Histórico com Inteligência

122 projetos de 5 anos digitalizados e indexados. O sistema compara automaticamente anos, identifica padrões, mostra quais clientes voltaram, quais segmentos cresceram, qual foi o melhor e pior mês. Isso é dado que a concorrência não tem — e que o fundador leva no bolso.

💡 Calculadora de Viabilidade em Tempo Real

Ao montar um orçamento, o sistema monta a equipe automaticamente baseado no pacote escolhido, calcula custo real, mostra margem esperada e alerta se o negócio é inviável antes de enviar a proposta. Isso evita aceitar projetos que dariam prejuízo — algo que acontece com frequência em produtoras sem controle.

📲 Zero Instalação, Zero Servidor

O sistema roda 100% no browser. Nenhuma instalação, nenhuma dependência de IT, nenhum servidor para manter. Isso elimina a barreira de adoção que destrói a maioria dos projetos de sistema interno.

Validação de Negócio

7. Validação como Ideia de Negócio

O problema que resolve

PMEs de serviços criativos (produtoras, agências, estúdios) no Brasil operam em um gap perigoso:

  • Muito pequenas para contratar ERPs como SAP, Totvs ou sistemas de gestão que custam R$2k–15k/mês.
  • Muito complexas para se contentar com planilhas Excel ou Notion (perdas de informação, duplicação, sem integração).
  • Sem profissional de TI interno para implementar e manter ferramentas genéricas.

Esse gap existe em centenas de milhares de empresas no Brasil que faturam entre R$500k e R$10M/ano.

8. Perfil de Clientes Ideais

Perfil de empresa que mais se beneficia

  • Faturamento: R$500k a R$10M/ano
  • Equipe: 2 a 30 pessoas (fixas + freelancers)
  • Setor: serviços criativos (audiovisual, marketing, publicidade, eventos, arquitetura, consultoria)
  • Dor atual: operando por planilhas + WhatsApp, fundador centralizando tudo, sem visibilidade de margem real
  • Maturidade digital: já usa ferramentas básicas mas não tem sistema integrado

Segmentos prioritários com análise de fit

🎬 Produtoras de Vídeo / Audiovisual FIT MÁXIMO

Exatamente o caso da AGE. Fluxo de produção (briefing → gravação → edição → entrega), catálogo de serviços com equipe variável, clientes recorrentes vs. esporádicos, NPS por etapa faz sentido total, fechamento financeiro com custo de freelancers é o maior ponto de dor.

📣 Agências de Marketing / Social Media FIT ALTO

Múltiplos clientes simultâneos, jobs mensais recorrentes, equipe mista, necessidade de mostrar ROI para o cliente. O módulo de NPS anônimo por etapa é um diferencial enorme — permite medir satisfação sem constranger o cliente.

🎪 Empresas de Eventos FIT ALTO

Orçamentos complexos com muitas variáveis (equipe, equipamento, local), fechamento financeiro pós-evento com gastos detalhados, pipeline de vendas (cotação → aprovado → realizado), histórico de clientes recorrentes.

🏛️ Arquitetura / Design de Interiores FIT MÉDIO-ALTO

Projetos longos com marcos, equipe de colaboradores externos, orçamentos por etapa, necessidade de CRM de clientes com histórico. Adaptação necessária: substituir "gravação" por "visita técnica", etc.

🧠 Consultorias e Treinamentos FIT MÉDIO

Pipeline de vendas, gestão de projetos por cliente, financeiro integrado. Menos aderência ao catálogo de pacotes com equipe variável — requer adaptação no módulo de orçamentos.

Sobre a plataforma — Genspark vs. Desenvolvimento próprio

O sistema atual roda 100% na Genspark. Para escalar como negócio de forma profissional, o caminho natural é:

  1. Curto prazo (0–6 meses): usar a Genspark para os primeiros clientes, validar o modelo, gerar caixa
  2. Médio prazo (6–18 meses): migrar para infraestrutura própria (ex: Vercel + Supabase) com Claude/Cursor para acelerar desenvolvimento
  3. Longo prazo (18m+): produto SaaS com painel de administração, permitindo que o próprio cliente personalize parâmetros sem necessidade de dev

A vantagem competitiva real não está na tecnologia — está no conhecimento de negócio do setor criativo, na capacidade de entender a dor do cliente e traduzir para um sistema que funciona no dia seguinte.
